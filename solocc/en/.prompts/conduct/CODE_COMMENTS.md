# Code Comments Standards

**Core Principle**: Comments are for other developers, not your personal notes.

## Prohibited

- Thinking process comments (e.g., "// First try this approach")
- Exploratory comments (e.g., "// Maybe this will work?")
- TODO comments without tracking
- Non-English comments

## Required

- Comments must be final, polished version
- Use English only
- Focus on "why", not "what"
